# 独立搜索窗口恢复修复设计

## 问题

`command-palette` 窗口虽然按 720×520 创建，但 `index.html#/command-palette` 没有可靠进入独立路由。窗口因此渲染了主布局，并跟随主窗口显示当前工具页面。

错误渲染出的主布局包含通用标题栏。用户点击关闭按钮后，前端调用窗口 `close()`；后端只阻止 `main` 窗口关闭，没有阻止 `command-palette` 关闭。搜索窗口被销毁后，全局快捷键只能查找已有窗口，查找失败便直接返回，所以无法再次唤起。

## 目标行为

- `command-palette` 始终渲染独立搜索页，不受当前 Hash 路由和主窗口页面影响。
- 快捷键在搜索窗口隐藏时显示它，在显示时隐藏它。
- 搜索窗口收到关闭请求时只隐藏、不销毁，因此之后仍可由快捷键唤起。
- 主窗口现有的关闭行为保持不变。

## 设计

1. 将搜索窗口配置地址改为 `#/command-palette`，使用 HashRouter 对应的直接路由形式。
2. React 入口同时读取当前 Tauri 窗口标签。当标签为 `command-palette` 时，强制选择 `/command-palette` 独立组件，即使地址异常也不渲染主布局。
3. Tauri 的窗口关闭事件单独识别 `command-palette`，调用 `hide()` 并阻止默认关闭。该判断放在主窗口关闭逻辑之前，不改变 `main` 的现有设置。

## 回归覆盖

- 配置测试要求搜索窗口使用 `#/command-palette`。
- 集成测试要求 React 根据 `command-palette` 标签选择独立页面。
- 集成测试要求后端对搜索窗口执行“隐藏并阻止关闭”。

## 范围限制

不修改搜索功能、主窗口导航、其他独立窗口或通用标题栏，不增加依赖。
